## Status Reports

All daily/weekly status reports must follow this filename pattern:

```
DD-MM-YYYY-daily-status.md
```

Example: `13-04-2026-daily-status.md`

This applies to all roles: Tech Lead, Dev 1, Dev 2, Dev 3, P.O.

## MVP (Completed 2026-04-12)

All planning, management, and team documentation for the MVP is archived in the `mvp/` folder:

- about.md - project goal and vision
- docs/architecture.md - technical architecture
- management/roadmap.md - sprint roadmap and kanban board
- management/kanban-sprint1.md through kanban-sprint6.md - sprint kanbans
- job/roles.md - role definitions
- job/job-roles.md - job descriptions
- mvp/features.md - all features (implemented)
- mvp/questions.md - Tech Lead <-> P.O. questions (all answered)
- mvp/dev-workflow.md - developer workflow guide
- mvp/test-plan-sprint1.md - test script
- mvp/v2-security-improvements.md - post-MVP security backlog
- reports/po/ - P.O. daily reports
- reports/tech-lead/ - Tech Lead daily/weekly reports
- reports/dev1/, dev2/, dev3/ - developer logs and questions
